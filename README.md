# パシャっと単語帳（flashcard-snap-kit）

単語帳の写真を、コーディングエージェント（Claude Code など）に渡すだけで、
スマホで使える「自己採点式の単語・例文テスト」に変換するための静的サイトです。
韓国語・英語などの外国語に加え、**古文・漢文**にも対応します。

> 🔗 **ライブデモ**: https://flashcard-snap-kit-demo.ayuphys.com
> （著作権物を含まない韓国語・英語・古文のサンプルだけが入っています）

## ⚠️ はじめに（必ずお読みください）

**著作権について（最重要）**
教科書・単語帳・参考書などの市販教材には著作権があります。本ツールは、それらを
**「自分の学習のためだけ」に使う**ことを前提に作られています。次の3つを守ってください。

1. **素材の写真をリポジトリに入れない。** 写真は `scans/` に置く（gitignore 済みなので、
   コミットも配信もされません）。
2. **生成したデータ（単語帳の JSON）を公開しない。** 使い方は「ローカルだけ」か
   「認証をかけたデプロイ」の2択。誰でも見られる場所には置かないでください。
3. **第三者に渡さない。** スキャン画像・生成データ・それらを含むリポジトリを、
   権利者の許可なく公開・配布・共有しないでください。

要するに——**自分で買った教材を、自分が勉強するためにデジタル化して使うだけ**なら
私的使用の範囲で問題ありません。ネットで公開したり他人に配ったりするのは NG です。

**入力データが AI の学習に使われない？**
写真やテキストを LLM に渡すので、「他人の教材が AI の学習に吸い込まれないか」が
気になるところです。鍵になるのは **どの経路で使うか** です。

- **開発者向け API・コーディングエージェント（Claude Code など）→ 比較的安心。**
  Anthropic・OpenAI などの API は「入力をモデルの学習に使わない」と規約で明記しています。
- **一般向けの Web チャット（無料版の ChatGPT など）→ 注意。** 既定で会話が学習に
  使われることがあるので、設定で **学習利用をオフ（オプトアウト）** にしてから使う。
- 扱いはサービスやプランで変わります。**使う前に各サービスの規約を確認**してください
  （本記載は法的助言ではありません）。

> 日本の著作権法 30 条の 4 は情報解析目的の AI 学習を広く認めていますが、著作者の心情や
> 海外の法律では議論が分かれます。迷ったら「**私的使用にとどめ、生成結果は人に渡さない**」
> を徹底するのが安全です。

## できること

- 単語帳の写真 → **単語テスト**・**例文テスト**・（必要なら）**文法テスト**を生成。
- **対応言語を問わない**。各ページが言語コード（`ko-KR`、`en-US` など）を宣言し、
  発音読み上げ（ブラウザTTS）にも使われます。
- **古文・漢文にも対応**（読み上げ＝音声はなし）。他の言語と同じく単語／例文で学べます。
- タップで答えを表示、シャッフル／並び順を戻す、前後ページ移動。
- 項目ごとに **未着手 / 得意 / 苦手** の3状態（◎得意・✕苦手）を記録。
  各ページ内と一覧ページに、単語・例文それぞれの **進捗（得意割合）を円グラフ**で表示。
- **苦手の復習**：全ページ横断で ✕（苦手）の項目からランダム出題（`review.html`）。
- 学習状態（未着手/得意/苦手）は**ブラウザ内に自動保存**。閉じても次回まで保持されます
  （端末ごと・サーバ不要。ローカル利用でも残ります）。
- ホーム画面に追加（PWA）で、iPhone でもアプリのように使えます。

## デモ

**ライブデモ**: https://flashcard-snap-kit-demo.ayuphys.com
著作権物を含まないサンプル（韓国語・英語・古文の3ページ）だけを公開した動作デモです。
パスワードなしで、そのまま触って試せます。

**手元で試す**: clone してルートで簡易サーバを立てるだけ。同梱のサンプルデータで
すぐ動きます（`file://` で直接開くと `fetch` がブロックされるため、必ずサーバ経由で）。

```
python3 -m http.server 8000   # → http://localhost:8000/
```

## 使い方

> **大原則**：自分の単語帳から作ったデータ（著作権物）は、**「A. ローカルのみ」**か、
> **認証保護をかけたクラウド**でしか使わないこと。

### 共通：まず自分のデータを用意する

1. **リポジトリを複製する。** ローカルだけで使うなら普通の `git clone` でOK。
   自分のデータを GitHub に置くなら必ず **private** リポジトリにする
   （`data/` が著作権物由来になるため public 不可。`fork` も public になるので使わない）。
2. **素材を置く。** 学習素材の写真を `scans/page-NNN.jpg` に置く
   （`scans/` は gitignore 済みで、コミットも配信もされない）。
3. **データを生成する。** コーディングエージェント（Claude Code 等）に
   「`docs/GENERATION_PROMPT.md` に従って `scans/` の画像を変換して」と依頼すると、
   `data/page-NNN.json` が作られ `data/index.json` に追記される。
   - 同梱のサンプル（`data/page-001..003.json`）は削除し、`data/index.json` の
     `pages` を空にしてから自分のページを `001` から採番する（詳細は
     `docs/GENERATION_PROMPT.md`）。

### A. ローカルだけで使う（推奨・最も安全）

PC／スマホの中だけで使う。どこにも公開しないので著作権上もっとも安全です。

1. リポジトリのルートで簡易サーバを起動（`file://` 直開きは `fetch` が
   ブロックされるので必ずサーバ経由）：
   ```
   python3 -m http.server 8000
   ```
2. ブラウザで `http://localhost:8000/` を開き、ページを選んで学習。
3. （任意）同じ Wi-Fi のスマホから使う：
   ```
   python3 -m http.server 8000 --bind 0.0.0.0
   ```
   で起動し、スマホで `http://<PCのLAN内IP>:8000/` を開く（LAN内限定・外部公開なし）。
   iPhone は Safari の「ホーム画面に追加」でアプリのように使える（PWA）。

> 💾 **得意/苦手はローカルでも保持されます**（ブラウザの localStorage に自動保存）。
> ただし保存は「同じブラウザ・同じ URL（ポート）」単位です。**毎回 `8000` など同じポート
> で開く**こと（ポートを変えると別の保存領域になり、空に見えます）。`localhost` と
> `127.0.0.1` と LAN内IP も別扱い。閲覧データの消去・プライベートウィンドウでは消え、
> 端末間では共有されません。

### B. クラウドに置きたい場合（各自で）

外出先からも使いたいなら、各自の責任でクラウドに置けます。純粋な静的サイトなので
どこでもホストできますが、**自分の生成データ（著作権物）を置くなら、ホスティング側の
認証保護が必須**です。

- **おすすめは Cloudflare Pages ＋ Cloudflare Zero Trust（Access）。** Pages に
  デプロイし、Zero Trust の Access アプリで「自分のメールだけ許可」にすれば、
  認証はダッシュボードだけで完結します（アプリ側のコードは不要）。
- **素の GitHub Pages は不可。** アクセス制御の仕組みがなく（private リポジトリでも
  公開ページは誰でも見られる）、著作権物の置き場所には向きません。
- ページ内 JavaScript の「パスワード入力」だけでは保護になりません（静的サイトは
  結局データを落とせるため）。**認証はホスティング／エッジ側**でかけること。
- **必ず検証**：デプロイ後にシークレットウィンドウ（未ログイン）で開いて
  **弾かれること**を確認してから、自分のデータを置く。

具体的な操作は各サービスの公式ドキュメントを参照してください（本ツールは特定の
ホスティングに依存しません）。`scripts/deploy.sh` は Cloudflare Pages へ直接
アップロードしたい人向けの補助スクリプトです（`scans/` は除外されます）。

## 古文・漢文への対応

古文・漢文にも対応しています。他の言語と同じく**単語／例文**で学べますが、
古典語は読み上げになじまないため **音声（🔊）はオフ**で表示されます。

> 出題データの作り方（白文・返り点・書き下しなどの細かい扱い）は、エージェントが読む
> `docs/GENERATION_PROMPT.md` 側にまとめてあります。

## データ形式（最小例）

```json
{
  "id": "001",
  "title": "助動詞の練習",
  "lang": "ja-JP",
  "tts": false,
  "modes": ["sentence"],
  "sentences": [
    {
      "id": "s1",
      "text": "京には見えぬ鳥なれば、皆人見知らず。",
      "translation": "都では見かけない鳥なので、誰も見て知らない。",
      "notes": [ ["ぬ", "打消「ず」の連体形"], ["なれ", "断定「なり」已然形"] ],
      "grammar": "前の「ぬ」は打消の連体形。完了「ぬ」と取り違えない。"
    }
  ]
}
```

## ディレクトリ構成

```
flashcard-snap-kit/
├── index.html              ページ一覧（data/index.json を読む）
├── test.html               テスト画面（?page=NNN）
├── review.html             苦手の復習（全ページ横断・ランダム出題）
├── manifest.webmanifest    ホーム画面追加用（PWA）
├── favicon.ico             サイトアイコン
├── assets/                 style.css / cards.js / app.js / review.js / progress.js / icon 各種
├── data/                   ページデータ（同梱はサンプル。自分のデータに置き換える）
│   ├── schema.json         ページJSONのスキーマ
│   ├── index.json          ページ一覧（メニュー）
│   └── page-001..003.json  サンプル（韓国語・英語・古文）
├── docs/
│   └── GENERATION_PROMPT.md エージェントに渡す生成プロンプト
├── scripts/
│   └── deploy.sh           Cloudflare Pages 直接アップロード（任意・scans/ を除外）
└── scans/                  （自分で作成）写真置き場・gitignore 対象・配信されない
```
